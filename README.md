### 브랜치 전략
<br>

- 처음에는 Front-end와 Back-end의 브랜치를 격리하는 전략을 사용하였습니다.
  - 각 개인 브랜치는 다음과 같은 명명 규칙을 지정하였습니다.
    - [부모_브랜치명]-[브랜치_생성날짜]-[이름_이니셜]
- 웹 프로토타입을 만든 후 기능 별로 브랜치 전략을 변경하였습니다.
  - develop : 모든 기능을 총괄하는 브랜치입니다.
  - develop-작업자-일자 : 담당자별 작업 브랜치입니다.
